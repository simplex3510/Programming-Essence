## Literal (리터럴, 상수)
```cs
Debug.Log(30);
```
Literal은 '문자 그대로의'라는 뜻을 가진 영단어에서 파생되어 사용되고 있다. <br>
즉, 리터럴이라고 하는 것은 작성되어 값이 변하지 않는 고정된 데이터를 말한다. <br>
위 코드 스니펫에서는 `30`이 그러하다.

또한 다음의 코드 스니펫에서도 다양하게 리터럴을 확인할 수 있다.
```cs
Debug.Log("Hello, World!");
Debug.Log('U');
Debug.Log(true);

string str = "Hello, World!";
char ch = 'U'
bool b = true;
```
`"Hello, World!"`, `'U'`, `true`처럼 작성할 수 있으나, 그 의미나 값을 변하게 할 수 없는 고정된 데이터를 **리터럴**이라고 한다.

`U`라는 고유한 문자가 의미하는 바, 혹은 값을 바꿀 수는 없다. <br>
마찬가지로 `true`의 의미나 값을 임의로 바꿀 수는 없다.

이때, 변치 않는 값을 저장하기 위한 변수를 의미하기 위한 이름인 '상수'와 리터럴을 의미하기 위한 이름으로써의 '상수'라는 명칭을 혼용하여 사용하기도 한다.

결론적으로, 변수나 상수에 저장되는 문자, 숫자, 불리언 등의 실제 값을 나타내는 모든 데이터의 원형을 리터럴이라고 한다.


## Magic Number (매직 넘버)
```cs
Debug.Log(30);
```
위 코드에서 `30`은 코드를 읽는 사람에게 어떤 의미인지 전달할 수 없다. <br>
즉, **의미를 알 수 없는 상숫값**을 매직 넘버라고 한다. <br>
