## Primitive Types (기본 자료형)
어떤 특정 언어에서 **기본적으로 지원하는 데이터의 표현 형태**를 기본 자료형이라고 한다. <br>
언어적 차원에서 미리 지원하고 있기 때문에 **빌트인 자료형(Built-In Type)**이라고도 한다. <br>

이때, 기본 자료형을 '하드웨어에서 직접적으로 이해하는 자료형'이라고도 표현할 때가 있는데, 이는 CPU가 이해할 수 있는 데이터의 표현을 직접적으로 언어 차원에서 사용할 수 있도록 미리 지원하기 때문이다.

예를 들어 '4바이트 크기의 정수형 데이터'를 CPU가 이해할 수 있다고 한다면, 이를 인간의 시선인 프로그래밍 언어적 표현으로 `int`로 나타낼 수 있다.

이렇듯 하드웨어(CPU)가 직접적으로 이해하는, 즉 컴퓨터가 이해할 수 있는 가장 자연스러운 데이터 표현 형태가 바로 기본 자료형이다.

## Data Size
`bit(비트)`는 0 또는 1을 표현하는 데이터의 최소 표현 단위이다. <br>
이러한 비트가 8개가 되어 `8 bit`가 되면, 이를 `1 Byte`로 표현할 수 있다. <br>
`Byte(바이트)`또한 기본적으로 비트의 집합이며, 각 비트마다 0 또는 1을 표현할 수 있다. <br>
따라서 바이트는 0부터 시작하여 255까지, 총 **256($2^8$)개의 수를 표현**할 수 있다.

바이트 이후의 표현으로는 1000과 가장 가까운 2진수인 1024($2^{10}$)단위 마다 계산하여 `1024 Byte = 1 KB`로 나타낼 수 있다. <br>
이후로도 `1024 KB = 1 MB`, `1024 MB = 1 GB` 등으로 표현될 수 있다.

## Integer Types (정수형)
프로그래밍 언어나 환경에 따라서 동일한 자료형의 이름이라 할지라도 크기가 다를 수 있다. <br>
C#을 전제로 설명하므로, C#에서의 정수형의 종류와 크기는 다음과 같다. <br>
정수형의 의미 그대로, 소수점이 있는 실수에 대한 값은 저장할 수 없다.

| 자표형의 이름 | 자료형의 크기 | Byte 단위 |
| :--- | :--- | :--- |
| byte | 8bit | 1Byte |
| short | 16bit | 2Byte |
| int | 32bit | 4Byte |
| long | 64bit | 8Byte |

```cs
// 사용 예시

// 인벤토리 용량을 나타냄.
byte inventoryCapacity = 255;

// 피해량을 나타냄
int Damage = 634612;
```

## Floating-Point Types (부동소수점 실수형)
C#에서의 실수형의 종류와 크기는 다음과 같다. <br>
이러한 실수형 자료형은 데이터를 표현할 때 항상 실수로서 표현된다.

| 자표형의 이름 | 자료형의 크기 | Byte 단위 |
| :--- | :--- | :--- |
| float | 32bit | 4Byte |
| double | 64bit | 8Byte |

```cs
// 효율이라는 시스템이 있을 때

 // 버프 계수 - 효율을 141%로 사용한다는 의미
float buffFactor= 1.41f;

// 디버프 계수 - 효율을 75...%만 사용할 수 있음을 의미
double debuffFactor = 0.7567856; 
```


**부동소수점**이라는 말이 붙는 이유는, 예를 들어 '$0.01$'이라는 실수가 있다면 과학적 표기법으로 $1 \times 10^{-2}$로 표현할 수 있다. <br>
이때 부동소수점의 의미는 **소수점(point)**의 위치가 고정되어 있지 않고, 수의 크기에 따라 좌우로 **떠다니듯(floating)** 조절되기 때문이다. <br>
`float`와 `double`형의 차이는 소수점 아래의 정밀도 계산이 얼마나 충분히 정확한가를 나타내기 위해 크기가 다르다는 차이가 있다. <br>
이 방식은 `IEEE 754 Standard`의 계산법에 의거하여 작동된다.

## Charater Type (문자형)
오직 하나의 문자를 저장하고 표현하기 위한 자료형이다. <br>

| 자표형의 이름 | 자료형의 크기 | Byte 단위 |
| :--- | :--- | :--- |
| char | 16bit | 2Byte |

```cs
char alphabet = 'a';
char number = '1';
char dash = '-'
```

해당 자료형을 사용하여 문자를 표현할 때는 `'a'`와 같이 작은 따옴표를 사용하여 표현한다. <br>

다른 언어(`C/C++`)에서는 `char`형이 'ASCII Code(8bit)'를 사용하지만, `C#`에서는 'Unicode'를 사용하므로 16bit로 표현한다.

## Boolean Type (불리언형)
`true`(참)과 `false`(거짓)을 표현하는 자료형이다. <br>

| 자표형의 이름 | 자료형의 크기 | Byte 단위 |
| :--- | :--- | :--- |
| bool | 8bit | 1Byte |

논리적으로 참(true)과 거짓(false)은 1비트만으로도 표현이 가능하다. <br>
하지만 컴퓨터의 CPU가 바이트(Byte) 단위로 처리하는 것이 훨씬 효율적이기 때문에 1바이트를 할당하여 사용한다. <br>

이때 `false`는 `0`에 대응되는 값이며, `true`는 `0`이 아닌 값에 대응된다. <br>
`C/C++`등 일부 언어의 경우에는 각 값을 서로 변환이 가능하게끔, 혼용하여 사용할 수 있으나, `C#`에서는 **불가능**하다.

## Negative Number (음수)
기본 자료형은 음수도 표현한다.

`byte`의 경우 8개의 비트로 이루어져 있고, 총 0~255의 표현 범위를 가진다. <br>
```cs
// 0 0 0 0 0 0 0 0 = 8bit
```

이때 한정된 크기에서 음수를 표현하기 위해서는 양수 표현의 범위를 줄일 수 밖에 없으며, 크기에서 절반씩 표현할 수 있게 된다. <br>
이때, **2의 보수 방식**이라는 것을 활용한다. <br>
따라서 값 `0`이 `0 0 0 0 0 0 0 0`으로 표현됨을 전제로 하기 때문에 0과 양수 범위 (0 ~ 127), 음수 범위 (-1 ~ -128) 모두 표현할 수 있다.

이때, 값 '127'은 다음과 같이 표현될 수 있다.
```cs
// '0' 1 1 1 1 1 1 1  =  127
// '?' 1 1 1 1 1 1 1  = -127
```
최상위 비트 하나가 남고, 그 아래 7자리를 활용하면 다시 127의 표현 범위를 확보할 수 있다.

이렇듯, 마지막 자리를 **'Sign Bit'**(부호 비트)로써 사용하여 음의 자리를 표현할 수 있다. <br>
```cs
// '1' 1 1 1 1 1 1 1  = -127
// '1' 0 0 0 0 0 0 0  = -128 (음의 0은 없으므로 음수로 표현)
// '0' 0 0 0 0 0 0 0  = 0
```

위에서 설명 했듯, **음의 0**은 없기 때문에 음수에서 표현범위가 1개 만큼 더 넓다.

## Signed / Unsigned Types (부호가 있는 / 없는 타입)
경우에 따라, 음수를 사용하지 않아 표현하지 않는 만큼을 양수로 더 표현하고 싶은 경우가 발생할 수도 있다.
이러한 자료형을 **Unsigned Type**이라고 한다.
