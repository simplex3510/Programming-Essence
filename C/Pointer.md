## [문제 1] 포인터의 기본 변수 조작

**다음 C언어 코드의 실행 결과로 올바른 출력 값을 작성하고, 그 이유를 메모리 구조와 연관 지어 설명하시오.**

```c
#include <stdio.h>

int main() {
    int a = 10;
    int *p = &a;
    *p = 20;
    
    printf("%d\n", a);
    return 0;
}

```

## [문제 2] 포인터 연산과 배열

**다음 코드에서 포인터 변수 `p`의 연산이 일어난 후, `printf`를 통해 출력되는 값은 무엇인지 기술하시오.**

```c
#include <stdio.h>

int main() {
    int arr[4] = {10, 20, 30, 40};
    int *p = arr;
    p++;
    
    printf("%d\n", *p);
    return 0;
}

```

## [문제 3] 주소에 의한 참조 (Call by Reference)

**다음 프로그램이 실행되었을 때 메인 함수의 변수 `x`의 최종 출력 값을 쓰고, 이와 같이 포인터를 매개변수로 활용할 때의 이점을 서술하시오.**

```c
#include <stdio.h>

void changeValue(int *num) {
    *num = 50;
} 

int main() {
    int x = 5;
    changeValue(&x);
    
    printf("%d\n", x);
    return 0;
}

```

## [문제 4] 포인터와 배열의 문법적 관계

**포인터 변수와 배열의 이름 관계를 다룬 보기 중 문법적으로 틀린 설명을 고르고, 왜 틀렸는지 이유를 설명하시오.**
*(단, `int arr[5]; int *p = arr;` 로 선언되어 있다고 가정합니다.)*

1. `arr[2]`와 `*(arr + 2)`는 내부적으로 완전히 동일하게 처리된다.
2. 포인터 변수를 활용하여 `p[1]`이라고 써도 `*(p + 1)`과 같으며, 이는 `arr[1]`의 값을 나타낸다.
3. `arr = p;` 연산을 통해 배열이 메모리상에서 가리키는 시작 주소를 다른 포인터의 주소로 유연하게 변경할 수 있다.
4. `&arr[0]`과 배열 이름 `arr`은 동일한 메모리 주소 상수를 나타낸다.

## [문제 5] 이중 포인터 (Pointer to Pointer)의 이해

**다음 코드는 이중 포인터를 활용하여 값을 변경하는 예제입니다. 최종 출력 결과를 쓰고, `dptr` 연산이 변수 `val`에 도달하는 과정을 논리적으로 설명하시오.**

```c
#include <stdio.h>

int main() {
    int val = 100;
    int *ptr = &val;
    int **dptr = &ptr;
    
    **dptr = 300;
    
    printf("%d\n", val);
    return 0;
}

```
