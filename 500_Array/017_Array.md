# Array (배열)

배열은 동일한 자료형의 데이터를 여러개를 묶어놓은 자료구조다.  
배열 내부에 있는 각 데이터를 **요소**(element)라고 한다.  

배열을 선언할 때, 배열의 **크기**(size)를 먼저 결정하고, 이후에는 크기를 변경할 수 없다.  

단, 각 요소(데이터)를 수정하는 것은 가능하다.  

배열의 문법은 다음과 같다.

```txt
(Type)[] (VarName) = new (Type)[(size)]
```

예를 들어, `int` 타입의 크기가 `4`인 배열 `arr`은 다음과 같이 선언할 수 있다.

```cs
int[] arr = new int[4];
```

## 배열이 필요한 이유

가령, 10개의 `userID`와 `userScore`를 입력받고 출력하는 코드가 있다.

```cs
int user1ID = int.Parse(Console.Readline());
float user1Score = float.Parse(Console.Readline());

// ...

int user10ID = int.Parse(Console.Readline());
float user10Score = float.Parse(Console.Readline());
```

이때 다음과 같은 문제점이 있다.

* **코드 중복**  
    만약 1만 개의 `userID`와 `userScore`를 받아야 한다면, 수많은 변수를 일일히 선언하고 입력을 받아야 한다.

* **코드 수정의 어려움**  
    동일하게 1만 개의 `userID`와 `userScore`가 있다면, 이를 개별적으로 접근해야 하므로 유지보수, 즉 코드 관리의 어려움이 발생한다.  
    가령, 10위까지의 랭킹을 보여주는 프로토타입에서 100위까지의 랭킹을 보여주는 것으로 기획이 바뀌었을 때, 각각의 변수를 일일히 접근해야 하므로 비효율적이다.

이러한 문제 외에도 여러 문제가 있을 수 있다.  
이를 위해 **배열**(array)이 존재한다.

이러한 배열을 사용하여 다음과 같이 앞선 코드를 개선할 수 있다.

```cs
int[] userIDs = new int[10];
float[] userScores = new float[10];

userID[0] = int.Parse(Console.Readline());
userScore[0] = float.Parse(Console.Readline());

// ...

userID[9] = int.Parse(Console.Readline());
userScore[9] = float.Parse(Console.Readline());
```

## 배열의 선언과 정의(초기화)

배열은 기본적으로 다음과 같이 선언과 동시에 초기화가 가능하다.  

```cs
// case1: 요소를 정의하여 배열 크기의 생략
int[] userIDs = new int[] { 56, 12, 25 };

// case2: 요소의 정의만 표현
int[] userIDs = { 56, 12, 25 };
```

위 2가지 경우 모두 요소의 개수를 알 수 있으므로, 크기를 명시하지 않더라도 컴파일러가 이를 자동으로 인식하여 처리해준다.

## 색인(Index)

색인은 `0`부터 시작하며, 이는 곧 배열의 첫 번째 요소가 `[0]`으로 표현될 수 있음을 의미한다.  

그 이유는 메모리의 **오프셋**(offset)이라는 개념에서 비롯된다.  
오프셋은 시작 기준 위치에서부터 **얼마나 떨어져 있는가**를 의미한다.

다음과 같은 코드가 있다고 가정한다.

```cs
int[] userIDs = { 56, 12, 25 };
// int형 배열 userIDs에, 대입(저장)한다, {56, 12, 25}의 요소값을
```

따라서 배열은 첫 번째 요소, 즉 `56`을 기준으로 하여 `0`만큼 떨어져 있으므로 `[0]`으로 표현하는 것이다.  
그러므로 `2`만큼 떨어져 있으면 `[2]`로 표현할 수 있다.

특히, 배열이 가지는 요소들은 메모리 공간에서 선형적으로 연속되게 존재한다.

## 요소 접근

배열을 나타내는 기호인 `[]`는 **첨자**(subscript) 연산자라고 한다.  
이러한 첨자 연산자 내부에는 요소의 위치를 의미하는 수인 **색인**을 위치시킬 수 있다.  

다음과 같이 접근이 가능하다.

```cs
int[] userIDs = { 56, 12, 25 };

// userIDs의 2번째 인덱스에 접근하여 대입함.
int userID = userIDs[2];
```

요소, 즉 데이터가 배열 내부에서 위치하는 순서는 왼쪽에서 오른쪽으로의 방향으로 나열된다.  
따라서 `56`이 첫 번째(`[0]`), `12`가 두 번째(`[1]`), `25`가 세 번째(`[2]`) 요소가 된다.  

# Multi-Dimensional Array (다차원 배열)

1차원 배열은 선형적으로 나열되어 있는 데이터들의 집합이었다.  

그렇다면 2차원 배열은, 그러한 1차원 배열이 나열되어 있는 면의 형태가 된다.  
또한 3차원 배열은 그러한 2차원 배열이 나열되어 있는 입체의 형태가 된다.

## 다차원 배열의 예시

가장 단적으로는 게임 테트리스(Tetris)가 그러하다.

이전의 그리드(격자)에 대한 예시가 있었다.  
코드는 다음과 같다.

```cs
int sizeRow = (가로 영역의 크기를 입력받음);
int sizeCol = (세로 영역의 크기를 입력받음);

for (int y = 0, y < sizeCol; y++)
{
    for (int x = 0; x < sizeRow; x++)
    {
        (그리드 1칸 출력);
    }

    (다음 줄로 이동);
}
```

이를 2차원 배열을 사용하여 나타내면 다음과 같다.  

```cs
int sizeRow = (가로 영역의 크기를 입력받음);
int sizeCol = (세로 영역의 크기를 입력받음);
int[,] Grids = new int[sizeRow, sizeCol];

for (int y = 0, y < sizeCol; y++)
{
    for (int x = 0; x < sizeRow; x++)
    {
        그리드_출력(Grids[x, y]);
    }
}
```

가령, 아무것도 없는 상태의 그리드가 있을 수도 있다.  
또한 장애물이 있어 건축물 건설에 방해가 되는 그리드가 있을 수 있다.  

이렇듯, 각 그리드의 상태값을 저장하기에 적절한 대상이 되는 것이 바로 2차원 배열이 될 수 있다.

2차원 배열을 선언하는 문법은 다음과 같다.

```cs
(Type)[,] (VarName) = new (Type)[(RowSize), (ColSize)];
```

이렇게 생성된 2차원 배열은 `RowSize × ColSize`개의 데이터를 가진다.

## 배열 선언과 동시에 대입

비어있는 그리드를 `0`, 장애물이 있는 그리드를 `1`로 전제한다
이때, `3 × 3`크기의 임의의 그리드를 정의한다면 다음과 같을 수 있다.

```cs
{
    { 0, 0, 0 },
    { 1, 0, 0 },
    { 1, 1, 1 }
};
```

가령 `2 × 3`크기의 건물을 해당 그리드에 건설하고자 한다면, 장애물 1칸이 겹쳐 건설이 불가능하다.  

이러한 그리드를 배열로 나타내면 다음과 같이 표현할 수 있다.

```cs
int[,] Grids = new int[3, 3] = {
        { 0, 0, 0 },
        { 1, 0, 0 },
        { 1, 1, 1 }};
```

이때 배열의 각 행렬 크기를 생략하여 나타낼 수도 있다.

```cs
int[,] Grids = new int[,] = {
        { 0, 0, 0 },
        { 1, 0, 0 },
        { 1, 1, 1 }};
```

가독성을 위해 다른 줄에 개행하여 작성하였으나, 위 코드를 한 줄로 이어 작성하더라도 문법적으로 문제가 되지 않는다.

## 중첩 `for`문과 2차원 배열

이러한 2차원 배열을 순회하며 출력할 수 있다.  
행과 열이 있어 각각을 순회해야 하므로, 2개의 중첩된 `for`문이 필요할 것이다.  

그러한 배열이 데이터로 저장되어 있을 때, 이를 적절히 게임에서 출력한다면 다음과 같은 코드가 필요하다.

```cs
int sizeRow = (가로 영역의 크기를 입력받음);
int sizeCol = (세로 영역의 크기를 입력받음);
int[,] Grids = new int[sizeRow, sizeCol];

for (int y = 0, y < sizeCol; y++)
{
    for (int x = 0; x < sizeRow; x++)
    {
        그리드_출력(Grids[x, y]);
    }
}
```

첫 번째로 안쪽(inside) `for`문이 3번 실행되면 게임 화면이 다음과 같을 것이다.

| 0 | 0 | 0 |
| -- | -- | -- |

이후 안쪽 `for`문을 탈출하면 바깥쪽(outside)의 `for`문 `y`값이 증가하여 다음 줄의 그리드를 출력할 수 있게 된다.  

다시 두 번째로 안쪽(inside) `for`문이 3번 실행되면 게임 화면이 다음과 같을 것이다.

| 0 | 0 | 0 |
| -- | -- | -- |
| **1** | **0** | **0** |

이를 한 번 더 반복하여 완전한 크기의 그리드와 각 그리드의 상태를 알 수 있다.

| 0 | 0 | 0 |
| -- | -- | -- |
| **1** | **0** | **0** |
| **1** | **1** | **1** |

## N-Dimensional Array

쉽게 생각하여, 2D 그래픽의 게임은 2차원 배열을 사용하고 있다고 비유할 수 있다.  
그렇다면, 3D 모델을 사용하는 3D 그래픽 게임은 3차원 배열을 사용하고 있다고 할 수 있다.

이렇듯, 4차원과 5차원 또는 그 이상의 배열도 필요에 따라서 선언하고 사용할 수 있다.

하지만, 데이터가 많아질수록 순회를 위한 `for`문의 개수가 비례하게 증가한다.  

가령 2D 그래픽 게임에서, $N^2$개의 데이터를 처리하기 위해 2개의 `for`문이 필요하다.  
3D 그래픽 게임에서는 $N^3$개의 데이터를 처리하기 위해 3개의 `for`문이 필요하다.

데이터 $N$의 수가 커지면 커질수록 연산의 양은 기하급수적으로 올라가며, 많아진 연산량이 성능의 저하로 이어질 수 있다.

이것이 바로 2D 게임보다 3D 게임이 더 무겁다고 표현하는 이유라고 할 수 있다.
