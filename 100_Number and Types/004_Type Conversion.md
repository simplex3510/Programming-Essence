# Type Conversion (형 변환)

Type Conversion, 즉 **'형 변환'**이란 어떤 자료형이 다른 자료형으로 바뀌는 것을 의미한다. <br>
컴파일러가 자동으로 자료형을 변환하는 **묵시적 변환(Implicit Conversion)**과 개발자가 직접 의도하여 자료형을 변환하는 **명시적 변환(Explicit Conversion)**이 있다. <br>

이때 항상 자유롭게 형 변환이 가능한 것은 아니다. <br>

## 묵시적 형 변환 (Implicit Type Conversion)
**'묵시적 형 변환'**은 기본 자료형 간의 변환을 컴파일러가 자동으로 처리해주는 것을 의미한다.  
이러한 묵시적 형 변환은 특별한 문법이 필요하지 않으며, 다음의 경우에 발생한다.
```cs
// 7FFF FFFF (16진수로 나타낸 num1)
int num1 = 2147483647;  // int의 최댓값 

// 0000 0000 7FFF FFFF (16진수로 나타낸 num1)
long num2 = num1;
```

`long`의 크기는 `8 Byte`이며, `int`의 크기 `4 Byte`보다 크다.  
따라서 `long`은 `int`가 표현할 수 있는 크기를 포함하여 더 많은 데이터를 표현할 수 있다.  
이때 `long num2 = num1`에서 `num1`이라는 값은 컴파일러가 자동으로 `long` 데이터로 판단하여 `num2`에 대입하는 것이다.

두 자료형 간의 묵시적 형 변환이 발생할 때, 데이터의 손실이 없어야만 한다.  
따라서 더 큰 자료형으로의 변환이 발생하는 것이다.

### Loss of Data (데이터 손실)

반대로, 크기가 큰 타입에서 작은 타입으로의 변환도 가능하긴 하다.  
하지만 반드시 값이 충분히 작아야 데이터 손실의 문제가 없다.
```cs
// 0000 0000 7FFF FFFF (16진수로 나타낸 num1)
long num1 = int.MaxValue;

// 7FFF FFFF  (16진수로 나타낸 num1)
int num2 = num1;

// C/C++에서는 가능, C#에서는 컴파일 에러
```
이는 언어에 따라 가능할 수도, 불가능할 수도 있다.  
이러한 변환의 경우가 항상 안전하지 않기 때문에 C#의 경우에는 불가능하다.

예를 들어, 그러한 문제 상황을 가정하면 다음과 같다.
```cs
// 7FFF FFFF FFFF FFFF (16진수로 나타낸 num1)
long num1 = long.MaxValue;

// FFFF FFFF (16진수로 나타낸 num1)
int num2 = num1;

// C/C++에서는 가능, C#에서는 컴파일 에러
```
이처럼 `7FFF FFFF`의 값이 손실되므로, C# 컴파일러는 항상 큰 타입에서 작은 타입으로의 형 변환을 제한한다.


### Loss of Precision (정밀도 손실)
이때, 다음처럼 동일한 크기의 자료형이 있다.
```cs
int numInt;       // 4 Byte
float numFloat;   // 4 Byte

long numLong;     // 8 Byte
double numDouble; // 8 Byte
```
가령, `int`에서 `float`으로 변환할 때 자료의 손실이 발생할 수도 있다.  
이유는 두 타입이 데이터를 저장하는 방식이 다르기 때문이다.

`int`는 32비트 전체를 숫자를 표현하는 데 사용한다.  

하지만 `float`과 32비트를 부호(1), 지수(8), 가수(23) 세 부분으로 나누어 관리한다.  
실질적으로 숫자의 유효 숫자를 담당하는 부분인 가수부가 23비트뿐이다.

`int`는 31비트(부호 비트 제외)만큼의 정밀도를 가질 수 있는데, `float`은 유효 숫자를 담는 크기가 23비트다.  
따라서 일정 수준 이상(23비트 크기 이상)의 큰 정수를 `float`으로 바꾸면, 뒷부분의 숫자를 다 담지 못하고 반올림하여 근사치로 저장하게 된다.

정리하면, 동일한 크기의 자료형이라 할지라도 경우에 따라 자료의 손실이 발생할 수도 있다.  
대표적으로 정수형을 실수형으로 변환하는 것이 그러하며, 이유는 저장 방식의 차이에서 기인한다.

### Promotion (승격)
승격은 연산 시 데이터 손실을 방지하고 CPU가 효율적으로 계산할 수 있도록 작은 타입을 더 큰 타입으로 자동 확장하는 규칙을 의미한다.  
이러한 규칙에는 다음과 같은 것이 있다.

* **정수 승격**  
    `byte`, `sbyte`, `short`, `ushort`, `char` 타입의 변수끼리 연산을 하면, 결과값은 무조건 최소 `int` 타입으로 변환된다.  
    이때, `int`는 하드웨어(CPU)가 가장 효율적으로 계산할 수 있는 타입이기 때문에 `int`를 최소 기준으로 변환하는 것이다.  

    예를 들면 다음과 같다.
    ```cs
    byte a = 1, b = 2;
    byte result1 = a + b;  // Compile Error
    int result2 = a + b;   // OK
    ```
    `a + b` 연산이 시작될 때, 각각의 변수가 가진 데이터가 `int`로 변환된다.  
    즉, 실질적으로는 `int(1) + int(2)`가 연산되는 것이다.  
    따라서 `byte`타입인 `result1`에 `int`형 데이터를 담을 수 없으므로 컴파일 에러가 발생한다.

* **이항 연산 승격**  
    서로 다른 타입이 연산될 때 더 넓은 범위를 가진 타입으로 변환된다.

    예를 들어 다음과 같다.
    ```cs
    int num1 = 3;
    double num2 = 0.141592;

    Debug.Log(num1 + num2);  // 3.141592 출력
    ```
    `num1`과 `num2`의 타입이 서로 다르고, `int`보다 `double`이 더 큰 크기를 가진다.  
    따라서 승격에 의해 연산의 결과인 `num1 + num2`의 데이터 타입은 `double`이 된다.

## 명시적 형 변환 (Explicit Type Conversion)
개발자가 의도적으로 형 변환을 강제하는 것이 바로 **'명시적 형 변환'**이다.

다음과 같이 어떠한 값이 충분히 작아 다른 타입으로 변환이 가능함을 개발자가 판단한 경우에 사용할 수 있다.
```cs
// 0000 0000 0000 0011 (16진수로 나타낸 num1)
long num1 = 3;

// 0000 0011 (16진수로 나타낸 num1)
int num2 = (int)num1;
```
이처럼 `(Type)VarName`과 같은 형태로 사용할 수 있다.

### 데이터 손실 : 실수에서 정수로의 형변환
실수형을 정수형으로 변환하면 소수점 이하의 수는 버려진다.
```cs
double num1 = 0.9;
double num2 = 1.9;

// result1 = 0 + 1 = 1
int result1 = (int)num1 + (int)num2;

// result2 = 0.9 + 1.9 = 2.8
double result2 = num1 + num2;
```

### 형 변환 불가 : 숫자와 불리언
모든 타입들이 서로 변환될 수 있는 것은 아니다.
```cs
int num = 10;

// Compile Error
bool bBoolean = (bool)num;
```
언어에 따라서, C/C++의 경우에는 '0인 값'을 `false`로, '0이 아닌 값'을 `true`로 간주한다.  
하지만 C#은 숫자 타입과 불리언 타입 사이의 형 변환을 허용하지 않는다.
