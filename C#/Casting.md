# 🎮 C# 상속 캐스팅 개념 강화 문제

### 1. 인벤토리 시스템과 안전한 다운캐스팅 (`is`, `as` 활용)

게임 인벤토리에는 여러 종류의 아이템이 섞여서 들어옵니다. 캐스팅 실패로 인한 프로그램 강제 종료를 막는 안전한 코드를 작성하세요.

* **조건:**
* 부모 클래스 `Item`을 정의하고, 이를 상속받는 자식 클래스 `Weapon`과 `Potion`을 만드세요.
* `Weapon` 클래스에는 `public void Swing()` 메서드가, `Potion` 클래스에는 `public void Drink()` 메서드가 독자적으로 존재합니다.
* `Main` 메서드에서 `Item[] inventory = new Item[3];` 배열을 생성하고 `Weapon`, `Potion`, `Weapon` 객체를 무작위로 할당하세요 (업캐스팅).
* `for`문을 통해 배열을 순회하며, **`is` 또는 `as` 연산자를 사용하여 안전하게 자식 타입으로 변환**한 뒤 각각의 독자적인 메서드를 호출하세요.

### 2. 컴파일러와 런타임의 차이 (에러 원인 분석)

다음 코드는 컴파일(빌드)은 정상적으로 성공하지만, 프로그램을 실행(Runtime)하면 치명적인 에러(`InvalidCastException`)를 발생시키며 종료됩니다.

```csharp
class Unit { }
class Hero : Unit { }
class Monster : Unit { }

class Program
{
    static void Main()
    {
        Unit myUnit = new Monster();
        Hero myHero = (Hero)myUnit; // 런타임 에러 발생!
    }
}

```

* **문제:**
1. 컴파일러가 이 코드의 명시적 형변환 `(Hero)myUnit`을 에러로 잡지 못하고 통과시킨 이유는 무엇일까요?
2. 실행 시점에 에러가 발생하는 원인을 참조 변수의 타입(이름표)과 실제 메모리에 생성된 인스턴스(실체)의 관계를 들어 설명하세요.

### 3. 업캐스팅된 객체의 멤버 접근 제한

객체를 부모 타입으로 바라볼 때, 메모리에 남아있는 자식 고유의 자원에 접근할 수 있는지를 묻는 문제입니다.

* **조건:**
* 부모 클래스 `Npc`를 만들고 `public string name = "NPC";` 필드를 선언하세요.
* 자식 클래스 `Blacksmith`(대장장이)를 만들고 `public void Repair()` 메서드를 추가하세요.
* `Main` 메서드에서 `Npc master = new Blacksmith();` 형태로 객체를 생성하세요 (업캐스팅).
* 이때 `master.Repair();`를 호출하면 컴파일 에러가 발생합니다. 실제 힙(Heap) 메모리에는 `Blacksmith` 객체가 온전히 올라가 있음에도 불구하고 이 메서드를 호출할 수 없는 이유를 설명하고, 올바르게 호출되도록 코드를 수정하세요.

### 4. 매개변수 다형성과 다운캐스팅 (스킬 시스템)

하나의 메서드로 다양한 타입의 대상을 처리하는 스마트한 시스템을 구현합니다.

* **조건:**
* 부모 클래스 `Creature`를 만들고, 이를 상속받는 `Player` 클래스와 `Monster` 클래스를 만드세요.
* `SkillManager` 클래스를 만들고 내부에 `public void CastSkill(Creature target)` 메서드를 정의하세요.
* 이 메서드는 매개변수로 모든 `Creature`를 다 받지만, 내부 로직은 캐스팅을 통해 분기됩니다:
* 대상이 `Player`라면 -> "플레이어에게 힐을 시전합니다." 출력
* 대상이 `Monster`라면 -> "몬스터에게 데미지를 입힙니다." 출력


* **주의:** 메서드 오버로딩을 사용하지 말고, 오직 `CastSkill(Creature target)` **하나의 메서드 안에서 형변환 연산자만을 이용해** 구현하세요.

### 5. 최상위 타입 `object`와 캐스팅

C#의 모든 클래스의 조상인 `object` 타입을 활용해 다양한 데이터를 묶고 풀어내는 문제입니다.

* **조건:**
* `object[] gameData = new object[3];` 배열을 선언하세요.
* 0번 칸에는 문자열 `"Stage 1"`, 1번 칸에는 정수 데이터 `100` (점수), 2번 칸에는 앞서 만든 `Weapon` 객체를 대입하세요.
* `for`문을 돌며 이 배열의 요소를 하나씩 꺼내어 원래의 타입으로 안전하게 되돌린(다운캐스팅) 후, 다음과 같이 출력 및 실행되도록 하세요.
* 문자열인 경우: "[텍스트] Stage 1" 출력
* 정수인 경우: 점수에 50을 더한 최종 점수 출력
* `Weapon`인 경우: 내부의 `Swing()` 메서드 호출
