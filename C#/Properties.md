## 🎮 C# 프로퍼티(Property) 개념 강화 문제

### 1. 캐릭터 HP 제한 프로퍼티 (데이터 검증)

`Character` 클래스를 만들고, 캐릭터의 체력을 관리하는 `HP` 프로퍼티를 구현하세요.

* **조건:**
* 실제 데이터가 저장될 `private int hp;` 필드(뒷받침 필드)를 선언합니다.
* `HP` 프로퍼티의 `set` 영역에서 입력받는 값(`value`)이 **0 미만이면 0으로, 1000을 초과하면 1000으로** 제한하여 필드에 저장되도록 하세요.
* `Main`에서 값을 `-50`이나 `2000`으로 설정해보고, 프로퍼티가 비정상적인 값을 어떻게 걸러내는지 확인하세요.



### 2. 읽기 전용 닉네임과 자동 구현 프로퍼티

게임 시작 시 한 번 정하면 바꿀 수 없는 닉네임과, 자유롭게 변경 가능한 레벨 시스템을 구현하세요.

* **조건:**
* 닉네임(`Nickname`)은 **자동 구현 프로퍼티**를 사용하되, 생성자에서만 초기화할 수 있고 외부에서는 값을 변경할 수 없는 읽기 전용(Read-Only)으로 만드세요.
* 레벨(`Level`)은 읽고 쓰기가 모두 가능한 자동 구현 프로퍼티로 만드세요. 기본값은 1입니다.
* 생성자를 통해 닉네임을 입력받아 초기화하고, 외부에서 닉네임을 변경하려고 할 때 컴파일 에러가 발생하는지 확인하세요.



### 3. 계산된 프로퍼티 (Computed Property)

캐릭터의 스탯창이나 전투력을 표시할 때 자주 사용되는 '계산된 프로퍼티'를 연습합니다.

* **조건:**
* `Player` 클래스에 자동 구현 프로퍼티로 `AttackPower`(공격력, `int`)과 `CriticalChance`(크리티컬 확률, `int`)를 만드세요.
* 별도의 필드 없이, 오직 `get`만 존재하는 **`CombatPower`(전투력, `int`) 프로퍼티**를 만드세요.
* 전투력의 계산 공식은 $공격력 \times (1 + \frac{크리티컬\ 확률}{100})$ 입니다. (정수 나눗셈으로 인해 소수점이 버려지지 않도록 형변환에 주의하세요.)



### 4. 무기 공격력 제한과 쓰기 전용 느낌의 제어 (`value` 활용)

게임의 밸런스를 무너뜨리는 버그나 핵을 방지하기 위해, 비정상적인 값의 입력을 차단하는 무기 클래스를 만드세요.

* **조건:**
* `Weapon` 클래스에 `private int damage;` 필드와 `Damage` 프로퍼티를 만드세요.
* 무기의 공격력은 절대 기존 공격력보다 **한 번에 50을 초과하여 증가할 수 없다**는 제한을 `set`에 보십시오. (예: 기존 damage가 10인데 100을 대입하려고 하면 "비정상적인 수치 변경 시도!"를 출력하고 기존 값을 유지)
* 정상적인 범위 내의 변경일 때만 필드에 값을 저장하세요.



### 5. 접근 제한자 변경 (`private set`)

외부에서는 마음대로 바꿀 수 없지만, 내부의 특정 메서드를 통해서만 안전하게 변경할 수 있는 프로퍼티를 구현합니다.

* **조건:**
* `Guild` 클래스를 만들고, 길드의 자금을 나타내는 `Gold` 프로퍼티를 만드세요.
* 외부에서 `guild.Gold = 10000;` 처럼 직접 수정할 수 없도록 **`public int Gold { get; private set; }`** 형태로 선언하세요.
* 대신 길드 클래스 내부에 `AddGold(int amount)` 메서드를 만들어, 이 메서드를 통해서만 자금이 추가되도록 구현하세요.
