# 🎮 C# 메서드 오버라이드(Override) 개념 강화 문제 (중상급)

### 1. 기본 몬스터의 공격 재정의 (`virtual`과 `override`)

모든 몬스터의 공통 행동을 정의하고, 각 자식 몬스터들이 자신만의 독특한 공격 방식을 구현하도록 합니다.

* **조건:**
* 부모 클래스 `Monster`를 만들고, `public virtual void Attack()` 메서드를 정의하세요. 이 메서드는 `"몬스터가 기본 공격을 합니다."`를 출력합니다.
* `Monster`를 상속받는 자식 클래스 `FireSlime`과 `Wolf`를 만드세요.
* `FireSlime`은 `Attack()`을 오버라이딩하여 `"화염 슬라임이 불을 뿜습니다!"`를 출력하게 하세요.
* `Wolf`는 `Attack()`을 오버라이딩하여 `"늑대가 물어뜯기 공격을 합니다!"`를 출력하게 하세요.
* `Main` 메서드에서 각각의 인스턴스를 생성하고 `Attack()`을 호출하여 부모의 메서드가 아닌 자식의 메서드가 실행되는지 확인하세요.

### 2. 다형성을 활용한 일괄 공격 시스템 (배열과 오버라이드)

오버라이딩의 진짜 가치는 서로 다른 객체들을 부모 타입으로 묶어서 한 번에 제어할 수 있다는 점에 있습니다. (앞서 배운 캐스팅 개념과 연결됩니다.)

* **조건:**
* 1번 문제에서 만든 `Monster`, `FireSlime`, `Wolf` 클래스를 그대로 활용합니다.
* `Main` 메서드에서 `Monster[] stageMonsters = new Monster[3];` 배열을 선언하세요.
* 배열의 각 칸에 `new Monster()`, `new FireSlime()`, `new Wolf()` 객체를 각각 할당하세요 (업캐스팅).
* **`for`문을 사용하여 배열을 순회하며 모든 몬스터의 `Attack()` 메서드를 일괄적으로 호출**하세요.
* **생각해볼 점:** 배열의 타입은 `Monster[]`이지만 왜 각자 다른 공격 메시지가 출력될까요? 형변환(`is`, `as`) 없이도 다형성이 작동하는 원리를 고민해 보세요.

### 3. 부모의 기능을 유지한 채 확장하기 (`base.Method()` 활용)

자식 클래스에서 부모의 기존 기능을 완전히 무시하는 것이 아니라, 부모의 기능을 먼저 실행한 뒤 자신만의 기능을 추가(확장)하는 논리를 구현합니다.

* **조건:**
* 부모 클래스 `PlayerCharacter`를 만들고 `public virtual void Rest()` 메서드를 정의하세요. 이 메서드는 `"체력을 10 회복합니다."`를 출력합니다.
* 자식 클래스 `Priest`(사제)를 만들고 `Rest()` 메서드를 오버라이딩하세요.
* `Priest`는 신성한 힘을 가졌기 때문에 휴식할 때 마나도 함께 회복됩니다.
* **`base.Rest();` 문법을 사용**하여 부모의 체력 회복 기능을 먼저 실행시키고, 그 아랫줄에 `"마나를 20 회복합니다."`라는 메시지가 추가로 출력되도록 구현하세요.

### 4. 오버라이딩과 오버로딩의 차이 분별 (서술 및 검증)

개념이 유사하여 학생들이 가장 많이 혼동하는 오버라이딩(재정의)과 오버로딩(중복정의)의 차이를 코드로 검증하는 문제입니다.

```csharp
class Weapon
{
    public virtual void Upgrade() 
    { 
        Console.WriteLine("무기를 기본 강화합니다."); 
    }
}

class LegendarySword : Weapon
{
    // 문제: 아래 두 메서드 중 '오버라이딩'과 '오버로딩'을 각각 구분하고,
    // 이 코드가 정상적으로 컴파일 및 작동하기 위해 필요한 키워드를 채워 넣으세요.

    public ______ void Upgrade() 
    { 
        Console.WriteLine("전설의 검이 화려하게 각성 강화됩니다!"); 
    }

    public void Upgrade(int scrollCount) 
    { 
        Console.WriteLine($"{scrollCount}개의 주문서로 연속 강화합니다."); 
    }
}

```

* **문제:** 빈칸을 채우고, 두 개념의 결정적인 차이점(매개변수 시그니처, 상속 관계 유무 등)을 주석이나 출력문으로 설명하세요.

### 5. 사운드 시스템 제어 (다형성 종합 응용)

게임 내 오브젝트들이 파괴될 때 각자 다른 효과음을 내는 시스템을 다형성 배열과 오버라이딩으로 설계합니다.

* **조건:**
* 부모 클래스 `GameObject`를 정의하고 `public virtual void OnDestroy()` 메서드를 만드세요. 기본 출력은 `"오브젝트 파괴음 재생"`입니다.
* 이를 상속받는 자식 클래스 `GlassBox`(유리 상자)와 `Bomb`(폭탄)을 만드세요.
* `GlassBox`는 `OnDestroy()`를 오버라이딩하여 `"쨍그랑! 유리 깨지는 소리 재생"`을 출력합니다.
* `Bomb`는 `OnDestroy()`를 오버라이딩하여 `"콰아앙! 폭발 소리 재생"`을 출력한 뒤, 자신만의 독자적인 메서드인 `public void ExplodeDamage()` (출력: "주변에 50의 광역 피해!")를 추가로 호출해야 합니다.
* `Main`에서 `GameObject[] sceneObjects` 배열에 두 자식 객체를 담고 `for`문으로 `OnDestroy()`를 호출하되, **`Bomb` 클래스인 경우에만 다운캐스팅을 활용해 `ExplodeDamage()`까지 연쇄적으로 호출**되도록 코드를 작성하세요.
