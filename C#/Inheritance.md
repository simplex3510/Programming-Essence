## 🎮 C# 상속(Inheritance) 개념 강화 문제

### 1. 기본 캐릭터 클래스 확장 (`protected` 접근 제한자)

모든 게임 캐릭터의 기반이 되는 부모 클래스를 상속받아 새로운 직업을 만들어 봅니다.

* **조건:**
* 부모 클래스 `Monster`를 만들고 `protected string name;`과 `protected int hp;` 필드를 선언하세요.
* `Monster` 클래스에 자식들이 공통으로 사용할 `public void ShowStatus()` 메서드를 만들어 이름과 HP를 출력하게 하세요.
* `Monster`를 상속받는 자식 클래스 `Slime`을 만드세요. `Slime` 내부에서 부모의 `name`과 `hp`에 직접 접근하여 각각 "초록 슬라임", "50"으로 초기화하는 메서드를 작성하세요.
* **핵심 질문:** `name`과 `hp`를 `private`으로 선언하면 자식 클래스에서 어떤 문제가 발생하나요?



### 2. 상속 관계에서의 생성자 호출 순서 (Memory Flow)

부모와 자식 클래스가 각각 생성자를 가지고 있을 때, 메모리에 어떤 순서로 올라가는지 추적하는 문제입니다.

* **조건:**
* 부모 클래스 `Item`을 만들고, 생성자에서 "부모: 아이템 데이터가 메모리에 로드되었습니다."를 출력하게 하세요.
* `Item`을 상속받는 자식 클래스 `Weapon`을 만들고, 생성자에서 "자식: 무기 인스턴스가 생성되었습니다."를 출력하게 하세요.
* `Main` 메서드에서 `new Weapon();`을 실행했을 때 콘솔에 출력되는 문장의 순서를 확인하고, **왜 부모의 생성자가 먼저 호출되어야 하는지** 상속의 메모리 구조 관점에서 설명하세요.



### 3. `base` 키워드를 이용한 부모 생성자 호출 (Constructor Chaining)

자식 객체를 생성할 때, 부모 클래스의 매개변수가 있는 생성자에 데이터를 전달하는 방법을 연습합니다.

* **조건:**
* 부모 클래스 `Npc`는 `string name`을 매개변수로 받는 생성자 딱 하나만 가지고 있습니다.
* `Npc`를 상속받는 자식 클래스 `Merchant`를 만드세요.
* `Merchant` 클래스의 생성자에서 `string name`과 `string shopName`을 입력받은 뒤, `base` 키워드를 사용하여 `name`은 부모 생성자에게 넘겨주고 `shopName`은 자신의 프로퍼티에 저장하도록 구현하세요.



### 4. 부모 타입 배열에 자식 객체 담기 (업캐스팅 맛보기)

상속 관계를 활용하면 서로 다른 자식 객체들을 하나의 부모 타입 배열로 묶어서 관리할 수 있습니다.

* **조건:**
* 빈 부모 클래스 `Skill`을 만드세요.
* `Skill`을 상속받는 `FireBall` 클래스와 `Heal` 클래스를 각각 만드세요.
* `Main` 메서드에서 `Skill[] skillSlot = new Skill[2];` 배열을 선언한 뒤, 0번 칸에는 `new FireBall()`, 1번 칸에는 `new Heal()` 객체를 대입해 보세요.
* **생각해볼 점:** 데이터 타입이 다른 `FireBall`과 `Heal`이 어떻게 같은 배열에 들어갈 수 있을까요?



### 5. 상속 제한과 봉인 클래스 (`sealed`)

더 이상 상속을 허용하지 않아야 하는 보안 및 설계상의 이유를 이해하는 문제입니다.

* **조건:**
* 게임 내에서 절대 시스템 구조가 바뀌면 안 되는 `GameSystem` 클래스를 만드세요.
* 이 클래스는 외부에서 상속받아 악의적으로 변형할 수 없도록 **`sealed` 키워드**를 사용해 봉인 클래스로 선언하세요.
* 다른 개발자가 `class HackSystem : GameSystem` 코드를 작성했을 때 어떤 에러 메시지가 발생하는지 확인하고 이유를 서술하세요.
