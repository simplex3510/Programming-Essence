# Operator (연산자)
연산자(Operator)는 데이터를 어떻게 처리할지 지시하는 약속된 기호라고 할 수 있다.  
이때 연산자는 피연산자, 즉 연산의 대상을 반드시 하나 이상 가져야 한다.

## Arithmetic Operator (산술 연산자)
산술 연산자는 일반적으로 수행하는 사칙연산을 수행하는 연산자를 의미한다.  
더하기와 빼기, 곱하기와 나누기를 수행하는 연산자다.
```cs
sbyte num1 = 2;
sbyte num2 = 10;

// result1 = 12
sbyte result1 = num1 + num2;

// result2 = 1111 1000 = -8
sbyte result2 = num1 - num2;
// result2 = 1111 1000 = 248
byte result2 = (byte)num1 - (byte)num2;

// result3 = 20
sbyte result3 = num1 * num2;

// result4 = 0
int result4 = num1 / num2;
// result4 = 0.0
double result4 = num1 / num2;
// result4 = 0.2
double result4 = num1 / (double)num2;
```

`result2`의 연산에서, `Unsigned Type`인 타입의 값을 연산했을 때 음수를 기대하면 안된다.  
이러한 경우에는 `Underflow`가 발생하여 의도하지 않은 값이 발생할 수도 있다.

또한 `result4`의 연산에서, 두 자료형이 모두 정수형일 경우에도 의도하지 않은 값이 발생될 수 있다.  
따라서 실수값을 기대하는 경우에는, 실수형 타입으로 형 변환을 명시하여 연산의 과정에서 실수값을 사용하도록 해야 한다.

## 증가/감소 연산자 (Increment and Decrement Operators)
증감연산자는 `++` 또는 `--`를 연산자로 사용하며, 하나의 피연산자를 가진다.  
연산의 결과로 `1`만큼을 증가시키거나 감소시키는 연산자이다.

이때, 연산자를 피연산자의 앞 혹은 뒤 중 어느 쪽에 위치시키느냐에 따라 다른 연산을 진행한다.

피연산자보다 먼저 위치한 형태를 **'Prefix (전위)'**, 나중에 위치한 형태를 **'Postfix (후위)'** 라고 한다.

```cs
int num = 10;

++num;  // prefix (전위 증가)
num++;  // postfix (후위 증가)

--num;  // prefix (전위 감소)
num--;  // postfix (후위 감소)
```

### 전위와 후위의 차이
전위와 후위에 대한 차이는 증감 연산의 순서가 달라진다는 점이다.

이를 증가 연산자를 기준으로 표현하면 다음과 같다.
| 전위 증가 연산자 | (전위 연산 결과) | 후위 증가 연산자 | (후위 연산 결과)
| :-- | :--: | :-- | :--: |
| `int num = 10;` | 10 | `int num = 10;` | 10 |
| `int result1 = ++num;` | 11 | `int result1 = num++` | 10 |
| `int result2 = num;` | 11 | `int result2 = num;` | 11 |

동일하게 `num`의 값을 `1`만큼 증가시키지만, 그 순서가 다르다.

이를 코드에서 다르게 표현하면 다음과 같다.
```cs
// Prefix (전위)
int num = 10;
num = num + 1;  // ++
int result1 = num;
int result2 = num;
```
전위 증감 연산자는 **어떤 연산을 수행하기 전에** 증가/감소 시키는 연산을 수행한다.  
즉, 대입 연산보다 먼저 수행되어 `num`의 값을 증가시키고 `result1`에 저장되는 것이다.

```cs
// Postfix (후위)
int num = 10;
int result1 = num;
num = num + 1;  // ++
int result2 = num;
```
반면 후위 증감 연산자는 **어떤 연산을 수행한 후에** 증가/감소 시키는 연산을 수행한다.  
즉, 대입 연산이 먼저 수행되어 `result1`에 값을 저장한 이후에 `num`의 값을 증가시킨다.

감소 연산자 `--` 또한 동일하게 작동하며, 사용법과 전위/후위에 대한 차이도 동일하다.

### 실수형에 대한 증감 연산
실수 자료형의 변수에도 증감 연산자를 사용할 수는 있다.
하지만, 증감 연산자는 주로 '정수 단위'로 사용되며 실수에 대해서는 잘 사용되지 않는다.

실수형 변수는 소수점 계산이 목적이다.  
즉, 연속된 값의 표현을 목적으로 하는 변수를 이산적인 연산인 증감 연산자를 사용하면 가독성과 정확성(정밀도) 측면에서 불리하다.

결론적으로 실수형에 대한 증감 연산은 **권고되지 않고 있다**.
