# Bitwise Operator (비트 연산자)

비트 연산자는 어떤 데이터를 다룰 때, 그 데이터의 2진수 표현에 대한 연산을 다룬다.  
일반적인 산술 연산자보다 속도가 매우 빠르며, 메모리를 아끼거나 하드웨어를 직접 제어할 때 유용하다.

비트 연산자는 2개의 피연산자를 가지므로 '이항 연산자'에 속한다.  
따라서 2개 데이터 각각의 *비트 패턴(Bit Pattern)을 비교하며 연산을 진행한다.  
> **비트 패턴 (Bit Pattern)**  
    비트 패턴이란, 비트들의 나열 그 자체를 의미한다.

즉, 다음 2개의 4비트 데이터가 있다고 가정하면,
```cs
// 9 = 1001
// 6 = 0110
```
`9`와 `6`의 각 비트 자리를 하나씩 비교하여 연산을 수행한다.
```cs
// 9 = | 1 | 0 | 0 | 1 | 
// 6 = | 0 | 1 | 1 | 0 |
```


## `&` (AND) Operator : 논리곱 연산자

```cs
byte op1 = 12;  // 0000 1100
byte op2 = 9;   // 0000 1001

int andResult = op1 & op2;  // ?
```

위 코드에서의 변수를 4비트로 간략화하면 다음과 같이 나타낼 수 있다.

| 변수명 | $2^3$ | $2^2$ | $2^1$ | $2^0$ |
| :-- | :--: | :--: | :--: | :--: |
| op1 | 1 | 1 | 0 | 0 |
| op2 | 1 | 0 | 0 | 1 |
| andResult | ? | ? | ? | ? |

`&`(AND) 연산은, 각 자리의 비트가 **모두** `1`의 값을 가질 경우에 `1`을 반환한다.  
그 외에는 항상 `0`을 반환한다.  
따라서 위 표의 `op1`과 `op2`의 연산의 결과로 `1000`가 출력된다.

| 변수명 | $2^3$ | $2^2$ | $2^1$ | $2^0$ |
| :-- | :--: | :--: | :--: | :--: |
| op1 | 1 | 1 | 0 | 0 |
| op2 | 1 | 0 | 0 | 1 |
| andResult | **1** | **0** | **0** | **0** |

값 `1000`을 10진수로 표현하면 `8`이 된다.

## `|` (OR) Operator : 논리합 연산자

```cs
byte op1 = 12;  // 0000 1100
byte op2 = 9;   // 0000 1001

int orResult = op1 | op2;  // ?
```

위 코드에서의 변수를 4비트로 간략화하면 다음과 같이 나타낼 수 있다.

| 변수명 | $2^3$ | $2^2$ | $2^1$ | $2^0$ |
| :-- | :--: | :--: | :--: | :--: |
| op1 | 1 | 1 | 0 | 0 |
| op2 | 1 | 0 | 0 | 1 |
| orResult | ? | ? | ? | ? |

`|`(OR) 연산은, 각 자리의 비트가 **하나라도** `1`의 값을 가질 경우에 `1`을 반환한다.  
그 외에는 항상 `0`을 반환한다.  
따라서 위 표의 `op1`과 `op2`의 연산의 결과로 `1101`가 출력된다.

| 변수명 | $2^3$ | $2^2$ | $2^1$ | $2^0$ |
| :-- | :--: | :--: | :--: | :--: |
| op1 | 1 | 1 | 0 | 0 |
| op2 | 1 | 0 | 0 | 1 |
| orResult | **1** | **1** | **0** | **1** |

값 `1101`을 10진수로 표현하면 `13`이 된다.

## `^` (XOR) Operator : 배타적 논리합 연산자

```cs
byte op1 = 12;  // 0000 1100
byte op2 = 9;  // 0000 1001

int xorResult = op1 ^ op2;  // ?
```

위 코드에서의 변수를 4비트로 간략화하면 다음과 같이 나타낼 수 있다.

| 변수명 | $2^3$ | $2^2$ | $2^1$ | $2^0$ |
| :-- | :--: | :--: | :--: | :--: |
| op1 | 1 | 1 | 0 | 0 |
| op2 | 1 | 0 | 0 | 1 |
| xorResult | ? | ? | ? | ? |

`^`(XOR) 연산은, 각 자리의 비트가 **서로 다를 경우에** `1`을 반환한다.  
그 외에는 항상 `0`을 반환한다.  
따라서 위 표의 `op1`과 `op2`의 연산의 결과로 `0101`가 출력된다.

| 변수명 | $2^3$ | $2^2$ | $2^1$ | $2^0$ |
| :-- | :--: | :--: | :--: | :--: |
| op1 | 1 | 1 | 0 | 0 |
| op2 | 1 | 0 | 0 | 1 |
| xorResult | **0** | **1** | **0** | **1** |

값 `0101`을 10진수로 표현하면 `5`가 된다.

## `~` (NOT) Operator : 부정 연산자

```cs
byte op1 = 12;  // 0000 1100

int notResult = ~op1;  // ?
```

위 코드에서의 변수를 4비트로 간략화하면 다음과 같이 나타낼 수 있다.

| 변수명 | $2^3$ | $2^2$ | $2^1$ | $2^0$ |
| :-- | :--: | :--: | :--: | :--: |
| op1 | 1 | 1 | 0 | 0 |
| notResult | ? | ? | ? | ? |

`~`(NOT) 연산은, 각 자리의 비트를 **반전 시킨 값**을 반환한다.  
즉, `0`은 `1`로, `1`은 `0`으로 반전시킨 값을 출력한다.  
따라서 위 표의 `op1`과 `op2`의 연산의 결과로 `1111 0011`가 출력된다.

| 변수명 | $2^7$ | $2^6$ | $2^5$ | $2^4$ | $2^3$ | $2^2$ | $2^1$ | $2^0$ |
| :-- | :--: | :--: | :--: | :--: | :--: | :--: | :--: | :--: |
| op1 | 0 | 0 | 0 | 0 | 1 | 1 | 0 | 0 |
| notResult | **1** | **1** | **1** | **1** | **0** | **0** | **1** | **1** |

값 `1111 0011`을 10진수로 표현하면 `-13`이 된다.

# Bit Shift Operator (비트 시프트 연산자)

비트 시프트 연산자는, 비트 패턴을 왼쪽 혹은 오른쪽으로 이동시키는 연산자다.

하드웨어(CPU)의 자체에서도 이러한 명령어(instruction)가 존재한다.  
따라서 다른 연산들보다 하드웨어 친화적인 연산자이므로 연산의 속도가 매우 빠르다.

## `<<` (left-shift) Operator

```cs
byte op1 = 12;  // 0000 1100

int leftShiftResult1 = op << 1;  // ?
int leftShiftResult2 = op << 2;  // ?
```

위 코드를 다음과 같이 나타낼 수 있다.

| 변수명 | $2^7$ | $2^6$ | $2^5$ | $2^4$ | $2^3$ | $2^2$ | $2^1$ | $2^0$ |
| :-- | :--: | :--: | :--: | :--: | :--: | :--: | :--: | :--: |
| op1 | 0 | 0 | 0 | 0 | 1 | 1 | 0 | 0 |

이때 위 코드는, `op1`의 비트 패턴을 각각 `1`만큼 왼쪽으로 이동한 값과 `2`만큼 왼쪽으로 이동한 값을 각각의 변수에 저장하는 코드이다.

| 변수명 | $2^7$ | $2^6$ | $2^5$ | $2^4$ | $2^3$ | $2^2$ | $2^1$ | $2^0$ |
| :-- | :--: | :--: | :--: | :--: | :--: | :--: | :--: | :--: |
| op1 | 0 | 0 | 0 | 0 | 1 | 1 | 0 | 0 |
| leftShiftResult1 | 0  | 0  | 0  | **1** | **1** | 0  | 0  | **0**  |
| leftShiftResult2 | 0  | 0  | **1** | **1** | 0  | 0  | **0**  | **0**  |

이때, 밀려나는 비트는 **버려지게 되고**, 새롭게 생긴 비트 공간에는 **`0`이 채워진다**.  
따라서 `leftShiftResult1`에는 `24`가 저장되고, `leftShiftResult2`에는 `48`이 저장된다.

### 2배 증가 관계

위 코드는 즉 다음과 같이 표현된다.
```cs
byte op1 = 12;
int leftShiftResult1 = 24;
int leftShiftResult2 = 48;
```

비트를 한 칸 왼쪽으로 이동하면, `12`에서 $2^1$만큼 곱한 `24`가 된다.  
비트를 두 칸 왼쪽으로 이동하면, `12`에서 $2^2$만큼 곱한 `48`이 된다.

즉 left-shift연산에서 $n$만큼 이동하면, 원래의 수에서 $2^n$배 증가함을 알 수 있다.

## `>>` (right-Shift) Operator

left-shift 연산과 마찬가지로, right-shift 연산도 유사한 작동 방식을 가진다.
```cs
byte op1 = 12;  // 0000 1100

int rightShiftResult1 = op >> 3;  // ?
int rightShiftResult2 = op >> 4;  // ?
```

위 코드는, `op1`의 비트 패턴을 각각 `3`만큼 오른쪽으로 이동한 값과 `4`만큼 오른쪽으로 이동한 값을 각각의 변수에 저장하는 코드이다.

| 변수명 | $2^7$ | $2^6$ | $2^5$ | $2^4$ | $2^3$ | $2^2$ | $2^1$ | $2^0$ |
| :-- | :--: | :--: | :--: | :--: | :--: | :--: | :--: | :--: |
| op1 | 0 | 0 | 0 | 0 | 1 | 1 | 0 | 0 |
| rightShiftResult1 | **0** | **0** | **0** | 0 | 0 | 0 | 0 | **1**  |
| rightShiftResult2 | **0** | **0**  | **0**  | **0**  | 0  | 0  | 0 | 0 | 0 |

따라서 `rightShiftResult1`에는 `1`이 저장되고, `rightShiftResult2`에는 `0`이 저장된다.

### 2배 감소 관계

위 코드는 즉 다음과 같이 표현된다.
```cs
byte op1 = 12;
int rightShiftResult1 = 1;
int rightShiftResult2 = 0;
```

비트를 세 칸 오른쪽으로 이동하면, `12`에서 $2^3$만큼 나눈 값인 `1`이 된다.  
비트를 네 칸 오른쪽으로 이동하면, `12`에서 $2^4$만큼 나눈 값인 `0`이 된다.

당연하게도, `/`연산자와 마찬가지로 몫만 남으며 나머지는 버려진다.

즉 righ-shift연산에서 $n$만큼 이동하면, 원래의 수에서 $2^n$배 감소함을 알 수 있다.

# Bit flag (비트 플래그)
비트 플래그(Bit Flag)는 이 비트 패턴의 각 비트 자리에 특정한 의미(상태)를 부여하여 사용하는 기술이다.  
정수형 변수의 각 비트를 하나의 스위치(깃발)처럼 사용하는 기법이다.  
가령, `1`이면 'On(참)', `0`이면 'Off(거짓)'을 의미하도록 한다.

예를 들어 다음과 같은 상태가 있다고 하자.

| 세력 | A | B | C |
| :--: | :--: | :--: | :--: |
| A | - | 적대 | 우호
| B | 적대 | - | 중립
| C | 우호 | 중립 | - |

이러한 경우에, 비트 플래그 기술을 사용할 수 있다.

각 1번째 비트는 '우호', 2번째 비트는 '중립', 3번째 비트는 '적대'라고 할 때, 다음과 같이 나타낼 수 있다.

```cs
[Flags] // 이 enum이 비트 플래그로 사용됨을 명시
public enum Relation      // enum은 '상수의 묶음'으로 생각하고 진행
{
    None      = 0b_0000,  // 000 (0)
    Friendly  = 0b_0001,  // 001 (1)
    Neutral   = 0b_0010,  // 010 (2)
    Hostile   = 0b_0100,  // 100 (4)
}
```

또는, 비트 시프트를 이용하면 다음과 같이 나타낼 수도 있다.
```cs
[Flags] // 이 enum이 비트 플래그로 사용됨을 명시
public enum Relation     // enum은 '상수의 묶음'으로 생각하고 진행
{
    None      = 0,       // 000 (0)
    Friendly  = 1 << 0,  // 001 (1)
    Neutral   = 1 << 1,  // 010 (2)
    Hostile   = 1 << 2,  // 100 (4)
}
```

위 상수를 활용하는 것은 세력간의 관계를 판단할 수 있는 가장 빠른 방법 중 하나다.

가령, 다음과 같이 사용할 수 있다.

```cs
// 1. 관계 설정 (표의 데이터 대입)
Relation relationAB = Relation.Hostile;  // A-B: 적대
Relation relationAC = Relation.Friendly; // A-C: 우호
Relation relationBC = Relation.Neutral;  // B-C: 중립

// 2. 관계 확인 (HasFlag 메서드 활용)
Console.WriteLine($"A와 B는 적대적인가? : {relationAB.HasFlag(Relation.Hostile)}"); // True
Console.WriteLine($"A와 C는 우호적인가? : {relationAC.HasFlag(Relation.Friendly)}"); // True

// 3. 비트 연산자로 확인 (전통적인 방식, 속도가 더 빠름)
bool isNeutral = (relationBC & Relation.Neutral) != 0;
Console.WriteLine($"B와 C는 중립인가? : {isNeutral}"); // True

// 4. 복합 상태 (예: 우호적이면서 중립적인 특수 상태)
Relation complexStatus = Relation.Friendly | Relation.Neutral; // 011 (3)
Console.WriteLine($"현재 상태: {complexStatus}"); // Friendly, Neutral 출력 ([Flags] 덕분)
        
// 5. 상태 변경 (중립을 끄고 적대를 켬)
relationBC = (relationBC & ~Relation.Neutral) | Relation.Hostile;
Console.WriteLine($"B와 C의 바뀐 관계: {relationBC}"); // Hostile
```

특히 세력의 수가 많아지거나 관계 종류(동맹, 교전, 무역 등)가 늘어날 때, 메모리 절약과 연산 속도 측면에서 매우 효율적이다.  

# Bit Mask (비트 마스크)
비트 마스크(Bit Mask)는 말 그대로 특정 비트들을 **가리거나(Masking)** 혹은 **드러내기** 위해 사용하는 이진수 패턴을 말한다.

얼굴에 마스크를 쓰면 눈만 보이게 하거나 입만 가릴 수 있다.  
이처럼, 비트 마스크를 데이터 위에 겹쳐서(연산해서) 내가 원하는 위치의 비트만 골라내거나 수정이 가능하다.

앞서 배운 비트 연산자를 도구로 사용하여, 특정 위치의 비트를 제어한다.

* 추출 (AND): 특정 비트가 1인지 확인하고 싶을 때 사용 (가장 많이 쓰임)
* 삽입/켜기 (OR): 특정 위치를 강제로 1로 만들 때 사용
* 삭제/끄기 (AND NOT): 특정 위치를 강제로 0으로 만들 때 사용
* 반전 (XOR): 특정 위치의 상태를 뒤집을 때 사용

## 예 : 8가지 상태를 가진 캐릭터
8비트(1바이트) 변수가 하나 있고, 각 비트가 캐릭터의 상태라고 가정한다.
```cs
byte charState = 0b_0000_0000;
```
그리고 `1010_0001`의 비트 패턴을 '중독, 무적, 이동 중' 상태로 전제한다.

이때 '무적(Invincible)' 상태인지 확인하고 싶다면, 무적 비트만 `1`인 마스크를 만든다.
```cs
// 마스크(Mask): 00100000 (무적 비트 위치만 1)
const byte invincibleMask = 0b_0010_0000;
```

이후 `&`연산을 통해서 상태를 확인할 수 있다.
```txt
  10100001 (현재 상태)
& 00100000 (무적 확인용 마스크)
--------------------------
  00100000 (무적 상태 확인)
```

# Bit Flag와 Bit Mask의 차이

비트 플래그와 비트 마스크는 서로 뗄 수 없는 관계다.  
하지만 '목적'과 '역할'에서 차이가 있다.

아주 쉽게 비유하자면, 플래그는 '**상태**'이고, 마스크는 그 상태를 다루는 '**도구**'라고 할 수 있다.

* **비트 플래그 (Bit Flag)**  
    비트 하나하나에 의미를 부여한 데이터 그 자체 (상태값)
* **비트 마스크 (Bit Mask)**  
    특정 비트를 뽑아내거나 조작하기 위해 씌우는 필터 (연산 도구)

따라서 다음과 같이 정리할 수 있다.

|구분|	비트 플래그 (Bit Flag)|	비트 마스크 (Bit Mask)|
| :--: | :-- | :-- |
| 비유 | 여러 개의 스위치가 달린 패널 | 구멍 뚫린 종이 (필터) |
의미 | "이 전등은 켜져 있는가?" 하는 **상태값** | "세 번째 전등만 보고 싶어" 할 때 쓰는 **도구**
주체 | 데이터의 **내용** (무엇인가?) | 데이터의 **조작** (어떻게 할 것인가?)
예시 | 0101 (독 걸림, 이동 중) | 0100 (이동 중인지 확인하기 위한 값)

즉, 각각은 **상태의 집합**과 **연산을 하는 행위나 그 필터값**을 의미한다.

이렇듯, 비트 플래그와 비트 마스크는 **엄연히 다른 개념으로 작동**한다.
 
하지만 실무에서는 엄격히 구분하지는 않고, 둘을 엮어 "**비트 마스크 기법을 사용하여 비트 플래그를 제어한다**"고 표현한다.

# 비트 연산자를 사용하는 이유
단적으로, **최적화**라고 말할 수 있다.

* **압도적인 속도**  
    CPU가 가장 빠르게 처리하는 연산
* **용량 최적화**  
    수천만 개의 객체 상태를 저장해야 할 때, bool 배열보다 메모리를 수십 배 적게 사용
* **네트워크 효율**  
    데이터를 주고받을 때 패킷 크기를 최소화

그 외에도 다양한 분야(알고리즘, 그래픽스 프로그래밍 등)에 맞는 이유가 존재한다.  
하지만, 다양한 분야에서 사용되는 만큼 그 이유 또한 매우 많기에 따로 수록하지는 않는다.
