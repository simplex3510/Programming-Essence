# Operator Precedence (연산자 우선순위)

연산자 우선순위는 여러 연산자가 있을 때 어떤 연산자가 먼저 실행되는지에 대한 순서를 나타내는 것이다.

다음은 연산자 우선순위를 정리한 표이다.  
우선순위 번호가 낮을수록(1번) 먼저 실행된다.

순위 | 연산자 유형 | 주요 연산자 예시 | 결합 방향
:--: | :-- | :-- | :--:
1 | 기본(Primary) / 후위(Postfix) | `()`, `[]`, `.`, `->`, `var++`, `var--` | →
2 | 단항(Unary) / 전위(Prefix) | `++var`, `--var`, `+`, `-`, `!`, `~`, `(type)`, `sizeof` | ←
3 | 산술 (곱셈류) | `*`, `/`, `%` | →
4 | 산술 (덧셈류) | `+`, `-` | →
5 | 비트 시프트 | `<<`, `>>` | →
6 | 관계(비교) | `<`, `>`, `<=`, `>=` | →
7 | 관계(동등) | `==`, `!=` | →
8 | 비트 AND | `&` | →
9 | 비트 XOR | `^` | →
10 | 비트 OR | `\|` | →
11 | 논리 AND | `&&` | →
12 | 논리 OR | `\|\|` | →
13 | 조건(삼항) | `?:` | ←
14 | 대입 | `=`, `+=`, `-=`, `*=`, `/=`, `%=` 등 | ←
15 | 쉼표 | `,` | →

## 연산자 결합 법칙

연산자 결합 법칙은 표현식 내에 우선순위가 동일한 연산자들이 나열되어 있을 때, 어느 방향부터 차례대로 계산할지를 결정하는 규칙이다.

### 왼쪽 결합성 (Left-to-Right)

대부분의 이항 연산자가 이에 해당된다.  
수식을 왼쪽 끝에서 오른쪽 방향으로 계산한다.

대표적으로 다음의 식이 그러하다.

```cs
int num = 2 - 8 + 7;

/* Order
1. int num = 2 - 8 + 7
2. int num = -6 + 7
3. int num = 1
*/
```

### 오른쪽 결합성 (Right-to-Left)

단항 연산자, 대입 연산자, 조건(삼항) 연산자가 이에 해당한다.  
수식의 오른쪽 끝에서부터 왼쪽 방향으로 계산한다.

대표적으로 다음의 식이 그러하다.

```cs
int a, b, c;
a = b = c = 5;  //  a = (b = (c = 5));
```

`a = (b = (c = 5))` 순서로 실행되어, 먼저 `c`에 `5`가 저장된다.  
이후 `c`의 값이 `b`로 대입, `b`의 값이 `a`로 전달된다.

## 확실한 순서의 제어: `()`의 사용

연산자 우선순위와 결합법칙을 염두하여 가장 확실하게 순서를 제어하는 방법은 **소괄호(`()`)를 이용하는 것**이다.  
이를 통해 **무엇이 먼저 계산되어야 하는지를 명확하게 표현**할 수 있어 가독성이 높아지고, 실수를 줄일 수 있다.

## Order of Evaluation (평가 순서)

연산자 우선순위와 평가 순서는 다르다.

연산자 우선순위는 **표현식이 어떻게 묶이는지**를 정하는 규칙이고, 평가 순서는 **그 묶인 표현식이 실제로 어떤 순서로 계산되는지**를 정하는 규칙이다.

예를 들어, 다음과 같은 코드가 있다고 가정한다.

```cs
if (1 + 1 == 2 || 3 + 2 == 5 && 1 + 2 == 2)
{
   // codes...
}
```

연산자 우선 순위에 따르면, `&&`가 `||`보다 높다.
따라서 위 코드의 조건식을 우선순위에 따라 괄호로 묶으면 다음과 같다.

``` cs
if ( (1 + 1 == 2) || ((3 + 2 == 5) && (1 + 2 == 2)) )
{
   // codes...
}
```

하지만, 실제 실행 시에는 **왼쪽에서 오른쪽으로 평가가 진행**된다.

1. `1 + 1 == 2`를 먼저 계산하고, 결과로 `true`가 도출된다.
2. 이때 `||` 연산자에서 단락 평가(Short-circuit evaluation)가 발생한다.
3. 결과적으로 우선순위가 높은 `&&` 영역은 **실행조차 되지 않고 건너뛰게 된다.**

따라서 복잡한 표현식은 가능한 한 나누어 쓰고, 순서가 중요하다면 중간 변수를 사용하거나 `()`로 의도를 명확히 드러내는 것이 좋다.

### Sequence Point (시퀀스 포인트)

시퀀스 포인트 또는 평가 순서 규칙에 의해 **"피연산자들은 왼쪽에서 오른쪽으로 평가된다"는 원칙**을 가진다.

시퀀스 포인트란, 프로그램 실행 중 이전 작업의 결과가 완전히 정리되고 다음 작업으로 넘어가기 전의 특정 지점을 의미한다.

대표적으로는 세미콜론(`;`)이 그러하다.

`&&`와 `||` 연산자는 그 자체로 일종의 '분기점' 역할을 한다.  
왼쪽 피연산자의 평가가 끝나는 시점이 하나의 시퀀스 포인트가 되며, 이 결과에따라 우항을 평가할지 말지 결정하는 제어 흐름이 생긴다.

우선순위가 아무리 높아도, 논리 연산자의 단락 평가와 좌측 우선 평가 원칙에 의해 실행 순서에서 밀리거나 제외될 수 있다.
