🎮 C# 객체지향 및 생애 주기 강화 문제
1. 플레이어 캐릭터 생성과 초기화 (기본 생성자)

Player 클래스를 만드세요. 플레이어는 name(string)과 level(int), hp(int)를 가집니다.

    조건:

        기본 생성자에서는 "플레이어가 생성되었습니다."라는 메시지와 함께 기본값(이름: "초보자", 레벨: 1, HP: 100)을 설정합니다.

        매개변수를 받는 생성자를 오버로딩하여 이름, 레벨, HP를 자유롭게 설정할 수 있도록 하세요.

2. 던전 보스 몬스터의 생성과 소멸 (종료자 확인)

BossMonster 클래스를 만드세요.

    조건:

        생성자에서 "보스 몬스터 {이름}이 등장했습니다!"를 출력합니다.

        종료자(~BossMonster())에서 "보스 {이름}이 처치되어 메모리에서 해제됩니다."를 출력합니다.

        Main 함수에서 BossMonster 객체를 생성하고, 객체에 null을 대입한 뒤 GC.Collect()를 강제로 호출하여 종료자가 호출되는 타이밍을 확인해 보세요.

3. 무기 강화 시스템 (생성자와 인스턴스 멤버)

Weapon 클래스를 만드세요. 무기는 생성될 때마다 자동으로 고유한 serialNumber를 부여받습니다.

    조건:

        static 키워드를 사용해 생성된 무기의 총 개수를 기록하는 변수를 만드세요.

        생성자에서 static 변수를 증가시켜 현재 무기의 순번을 serialNumber에 저장하세요.

        5개의 무기를 생성하고 각 무기의 고유 번호를 출력하세요.

4. 포션의 유효 기간과 객체 참조 (종료자의 이해)

Potion 클래스를 만드세요.

    조건:

        생성자에서 potionID를 매개변수로 받습니다.

        특정 메서드 내에서 지역 변수로 Potion 객체를 생성하여 사용한 뒤, 메서드가 종료될 때 해당 객체가 어떤 상태가 되는지 고민해 보세요.

        종료자에서 "포션 {potionID}가 소멸되었습니다."를 출력하여, 객체가 더 이상 참조되지 않을 때 메모리 관리가 어떻게 이루어지는지 관찰하세요.

5. 게임 씬(Scene) 매니저 (객체지향 설계)

Scene 클래스를 만드세요.

    조건:

        생성자에서는 "씬 {씬 이름} 로드 중..."을 출력하고, 종료자에서는 "씬 {씬 이름} 데이터 정리 완료."를 출력합니다.

        Main에서 Scene 객체를 배열로 여러 개 생성하고, for문을 사용하여 차례대로 참조를 끊어보며 종료자가 호출되는 순서를 추측해 보세요. (FIFO 또는 LIFO 구조와 연관 지어 생각하기)
